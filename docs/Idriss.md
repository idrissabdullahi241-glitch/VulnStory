## One-Paragraph Description

VulnStory is an AI-powered interactive cybersecurity training application that transforms OWASP Top 10 vulnerabilities into immersive, story-driven scenarios. Users step into the role of attacker or defender inside realistic company environments — fintech platforms, hospital systems, e-commerce backends — and make real incident response decisions with immediate AI-generated feedback. Built on the Anthropic Claude API, every scenario is dynamically generated, ensuring no two sessions are identical. VulnStory bridges the gap between textbook security knowledge and real-world threat handling, making professional-grade security training accessible, engaging, and repeatable.

---

## The Problem We Are Solving

Cybersecurity education has a hands-on gap. Most training materials teach vulnerabilities as definitions — "SQL Injection is when user input is inserted into a SQL query without sanitization." What they fail to provide is the lived experience of encountering, recognising, and responding to an attack in progress.

Existing platforms like TryHackMe and HackTheBox are excellent but require significant setup, environment configuration, and time investment. Beginners and intermediate students often drop off before they reach meaningful learning moments.

VulnStory solves this by making scenario-based security training instant, AI-generated, and accessible entirely in a browser — no setup, no VM, no prior environment required.

---

## The Solution

VulnStory uses the Anthropic Claude API to generate unique, realistic vulnerability scenarios on demand. Each scenario includes:

- A realistic company setting (e.g. a fintech platform processing $2M in daily transactions)
- A situation description showing what the attacker or defender observes
- Real log snippets and code examples embedded in the story
- A subtle technical clue for the player to identify
- A decision point with four choices (only one correct)
- Immediate AI-generated feedback explaining the outcome and the correct defence technique
- An OWASP mapping for every scenario

The app supports three modes:

**Story mode** — Role-play as attacker or defender across 10 OWASP Top 10 vulnerabilities at beginner, intermediate, or advanced difficulty. Every scenario is AI-generated and unique per session.

**Quiz mode** — Situational multiple-choice questions generated by AI, testing applied knowledge rather than rote memorisation.

**OWASP Reference** — A built-in reference panel covering all OWASP Top 10:2021 entries with severity ratings, descriptions, and associated vulnerability types.

---

## What Makes It Creative

Most AI-assisted apps in this category generate text or images. VulnStory uses AI to generate *interactive decision trees* — structured scenario objects that drive a stateful UI. The AI output is not displayed as raw text; it is parsed, validated, and rendered as an interactive experience with choice buttons, outcome reveals, score tracking, and feedback panels.

The creative angle is the intersection of two domains that rarely meet: narrative storytelling and cybersecurity incident response. By wrapping technical security concepts in a story format — with a named company, a named attacker pattern, a timestamp, and real log data — the learning becomes contextual and memorable rather than abstract.

## Technical Implementation

### Stack

- Pure HTML5 / CSS3 / Vanilla JavaScript — no framework dependencies
- Anthropic Claude API (`claude-sonnet-4-20250514`) via direct browser `fetch()` calls
- Structured JSON output enforced through system prompt engineering
- Tabler Icons webfont for UI iconography

### API Integration

The app makes real-time calls to the Anthropic `/v1/messages` endpoint. A carefully engineered system prompt instructs the model to return only a valid JSON object with specific fields (title, setting, situation, clue, choices, outcomes, defence technique, OWASP ID). The app parses this JSON and renders each field into the appropriate UI component.

```
System prompt approach:
- Role: "You are VulnStory, an interactive cybersecurity training engine"
- Constraint: "Respond ONLY with valid JSON, no markdown, no preamble"
- Schema: Explicit field definitions with types and character limits
- Grounding: Vulnerability name, OWASP ID, role, difficulty injected per request
```

### Prompt Engineering

The story generation prompt injects four dynamic variables per request:

1. Vulnerability type (e.g. SQL Injection)
2. OWASP category (e.g. A03:2021)
3. Role (attacker or defender)
4. Difficulty (beginner / intermediate / advanced)

This produces scenarios that are contextually accurate, difficulty-calibrated, and role-appropriate — with no additional fine-tuning or training required.

### Pre-built Advanced Scenarios

In addition to the AI-generated content, the app ships with three hand-authored advanced SQL Injection defender scenarios that demonstrate domain depth:

- The phantom admin login — classic comment-based authentication bypass
- The exfiltration UNION — UNION-based data extraction against a medical records platform
- The blind boolean time-bomb — timing side-channel attack with no visible error output

These showcase the security domain knowledge behind the project, not just the AI integration.
